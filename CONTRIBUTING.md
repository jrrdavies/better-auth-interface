# Contributing to better-auth-interface

Thanks for your interest in contributing! This guide will help you get started.

## Development setup

1. Fork and clone the repo
2. Install dependencies:
   ```bash
   pnpm install
   ```
3. Verify everything works:
   ```bash
   pnpm typecheck && pnpm lint && pnpm format:check
   ```

## Adding a component

1. Create your component in `registry/auth/` or `registry/admin/`
2. Add an entry to `registry.json`
3. Run `pnpm build:registry` to generate the output
4. Test locally in a fresh project:
   ```bash
   npx shadcn add ./public/r/<your-component>.json
   ```

See the **Important rules** section below for code conventions.

## Pull request process

1. Create a branch from `main` (`feat/...`, `fix/...`, etc.)
2. Make your changes following the code conventions below
3. Run all checks before pushing:
   ```bash
   pnpm typecheck && pnpm lint && pnpm format:check
   ```
4. Open a PR with a clear description of what and why
5. Link any related issues

## Commit messages

Use [conventional commits](https://www.conventionalcommits.org/):

- `feat:` — new feature or component
- `fix:` — bug fix
- `docs:` — documentation changes
- `chore:` — tooling, config, CI changes
- `refactor:` — code restructuring without behavior change

## Important rules

- **Never import from `better-auth` directly** in registry components — use the auth-provider context
- **Never modify files in `src/components/ui/`** — these are generated by shadcn and must remain untouched
- **All dependencies must be MIT-compatible** — verify before adding new packages
- **Run `pnpm build:registry`** after any component changes to regenerate `public/r/`

## Reporting bugs

Use [GitHub Issues](https://github.com/jrrdavies/better-auth-interface/issues) with the bug report template.

## Suggesting features

Open an issue using the feature request template. Describe the use case and how it relates to Better Auth.
